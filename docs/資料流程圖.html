資料層級（誰裝在誰裡面）

日期資料夾 ← 只是「記錄按日期分組」的視圖，不是另一份輸入 └ 流水號 = 圖號 = 一張修改單（ChangeOrder） └ 焊口（碼 + 尺寸 / 材質 / 厚度） └ 照片（修改前 / 修改後） └ 圖面 PDF └ 材料 └ 狀態

關鍵：狀態住在「流水號」這層，不是日期。一個日期可能 3 張圖號只產了 1 張 → 日期顯示聚合「已產 1/3」。

分頁職責（產出 vs 管理）

分頁角色做什麼
產出報告做挑日期 / 圖號 → 開始執行 → 產出 報告 / PDF；跑完回寫狀態
紀錄管理看 & 改瀏覽每個流水號的焊口明細 / 材料 / 狀態，編輯、重產、開資料夾
材料價目單價庫提供單價 → 算金額，餵給 紀錄管理 / 請款
請款追蹤彙總焊口金額 + 材料金額 → 請款
健康檢查來源 ↔ 記錄 一致性檢測

狀態流轉

草稿 → 待補（缺照片 / PDF）→ 完整 → ＜產出報告跑完＞ → 已產出　/　有錯

一句話：建立靠「修改單精靈」，真相存在「ChangeOrder 記錄」，其餘分頁都圍著這份記錄打轉。
