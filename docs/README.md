# 文件中心

這裡放專案產品化、架構、討論與決策文件。

## 最高優先入口

- `00_big_bang_產品進化總控室/`

這個資料夾是公司級產品化的總控室。後續如果有多個 AI、不同模型或不同回合進來討論，請先從這裡讀起，不要直接從零發散。

## 文件分工

- Opus 前導書：大方向、產品野心、風險清單，視為核心原始藍圖。
- 決策紀錄：只放已收斂的選擇，不放發散廢稿。
- Phase 開發板：把藍圖拆成可以實作、驗收、測試的任務。
- AI 討論紀錄：保留不同 AI 的提案、批判與後續整理線索。
- inbox：先收還沒整理的草稿、截圖、外部建議。
