# UI 現代化設計規範：深色卡片可出貨版

> 校準者：Opus｜日期：2026-06-22｜對象：工務修改單 GUI 視覺現代化
> 此文件為設計規範，刻意不提交，待人類 review 後自行 commit。
> 目標氛圍參考：`docs/00_big_bang_產品進化總控室/inbox/工務修改單_交付中心_glass_mockup.html`（這是 mood，不是 1:1 還原目標）。

## 決策（已由使用者拍板）

- **風格：深色卡片現代化（可出貨）。** 追求 PyQt 能穩穩做到的現代感，約等於參考圖 70%。
- **不做：** 真毛玻璃（backdrop blur）、Win11 acrylic/mica、frameless 自繪視窗、3D 浮動輪播、動畫過場。
- **時機：RC0 內測之後才動手**，排在 Checkpoint G 的三個 P0（樣板路徑 / Excel COM / gate 真實產出冒煙）之後。先止血、先讓內測跑完，再美化。

## 邊界：為什麼不做真毛玻璃（先講清楚，免得 Codex 亂衝）

- Qt 沒有原生 `backdrop-filter`。真毛玻璃只能：(a) Win11 DWM acrylic——平台綁定、版本脆、多機不可靠；(b) 截背景圖再 `QGraphicsBlurEffect`——吃效能、會閃。內部工具要穩，兩條都不採。
- frameless（`FramelessWindowHint` + 自繪標題列）會失去原生 snap / 拖曳縮放 / 高 DPI 行為，要全部自己重刻，是 bug 與維護黑洞。**v1 保留原生視窗框，只重塗內容。**
- 所以「玻璃感」用**深色半透明卡 + 細邊 + 強調色 + 克制的陰影**去逼近，不是真的模糊。

## 設計 token（落在既有 `control/theme.py` 的 `Colors`，新增 Dark 變體，不要另開新檔）

現況：`theme.py` 已有集中的 `Colors` 類（目前是淺色：bg `#f8fafc`、card `#ffffff`、primary `#2563eb`、accent `#7c3aed`）。**深色現代化主要就是動這個集中檔**，不必拆 `gui_panels.py`。新增一組 dark token：

表面（由深到淺，營造層次）：

- `BG`（視窗底）：`#0E1422`
- `BG_PANEL`（側欄／分頁底）：`#161E2E`
- `BG_CARD`（卡片）：`#1C2638`；hover `#222E44`
- `BG_ELEVATED`（彈窗／下拉）：`#243049`

邊框與分隔：

- 預設 `rgba(255,255,255,0.08)`；hover `rgba(255,255,255,0.14)`；強調邊 `rgba(54,214,176,0.35)`

文字：

- 主 `#E8EEF7`；次 `rgba(232,238,247,0.66)`；提示 `rgba(232,238,247,0.42)`；disabled `rgba(232,238,247,0.30)`

強調與語意：

- 主強調（主要動作）`#36D6B0`，hover `#54E3C4`，按鈕上文字 `#06241D`
- 次強調（選取／focus 環）紫 `#8E8CFF`
- 語意：success `#46C28C`、warning `#E0A33A`、danger `#E2574E`、info `#4F9FE0`

狀態 pill（對應交付中心）：

- 已輸出＝teal、缺照＝amber、未入庫＝violet、失敗＝red、略過＝gray

形狀與節奏：

- 圓角：卡 14px、輸入框/按鈕 10px、pill 999px、彈窗 16px
- 間距：8 / 12 / 16 / 24 四級
- 字級：base 13px、caption 11px、卡標題 15px、區標題 18px；**字重只用 400 與 500**；字型 `"Microsoft JhengHei","Noto Sans TC",sans-serif`
- 陰影：Qt 沒有 widget box-shadow，只能用 `QGraphicsDropShadowEffect`，**僅用在焦點卡這種少數元件**（每個 effect 都有重繪成本，不可全鋪）。層次主要靠「底色對比 + 細邊」撐。

狀態規則：hover＝底色微亮；selected＝左側 2px teal 邊或 teal 外環；focus＝紫 1px 環；disabled＝0.4 透明。

## 落地切片（RC0 後，安全順序）

- **前置：** 確認要動的畫面有 UI 煙霧測試（已有 MainWindow 與輸出中心 smoke，見 doc 55、60）。沒測試覆蓋的畫面不要動。
- **切片 0　硬編碼顏色稽核（必做，否則深色主題會有亮斑）：** `grep -rnE "#[0-9a-fA-F]{6}|white|#fff" control/gui_*.py` 找出沒走 `Colors` 的色值，先收斂進 `Colors`。這步是純清理、有測試保護。
- **切片 1　深色主題層：** 在 `theme.py` 加 dark token + 一張全域 QSS，於 app 啟動套用（`app.setStyleSheet(...)`）。**零結構變更、零邏輯變更。** 視覺收益最大、風險最低。驗收＝煙霧測試綠 + 前後截圖對照。
- **切片 2　元件重塗：** 卡片、狀態 pill、按鈕、`QTableView`（斑馬列／表頭／選取列）、scrollbar。仍不動結構。
- **切片 3　交付中心升級：** 把「輸出中心」從 `RecordManagerPanel` 的彈窗升成第一級分頁（呼應 4 區工作流 IA）。這是結構變更、風險最高，**放最後、在測試網之後**。
- **明確不做（寫死禁止 Codex 自行展開）：** frameless、acrylic、真毛玻璃、動畫、3D 輪播、自繪標題列。

## 驗收與風險

- **每個切片都要前後截圖對照**——UI 改動沒有截圖等於沒驗收。
- **最大風險＝硬編碼色殘留**：任何沒走 `Colors` 的 `#ffffff`／`white` 在深色主題下會變亮斑。切片 0 沒做乾淨，切片 1 就會花。
- **對比度**：深底淺字要過 WCAG AA（正文 ≥ 4.5:1），別為了好看犧牲現場可讀性。
- **高 DPI / 多機**：公司機器解析度不一，QSS 用相對單位、避免寫死像素字級。
- **可回退**：主題層是一張 stylesheet，出問題能一行關掉回淺色——這也是「主題層先行」最安全的理由。
