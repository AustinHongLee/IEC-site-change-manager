# 新精靈 完整規劃（v0.1，討論用）

> **目的**：在繼續寫程式契約前，把「新修改單精靈」當一個整體重新對齊。第一波 thin-shell 雖然引擎對，但 UI 偏離了當初設計——**源頭驅動挑焊口被降成「打 base」、歷史 / 標註 / staging 側欄全缺**。這份把「它該長什麼樣」講清楚，你圈掉 / 加上，對齊後再回去切契約。
> **引擎不動**：`change_order` / `weld_lookup` / `weld_codec` / `change_order_builder` / `change_order_store`（都獨立驗過）是資產。這份只重畫「臉跟手」。

---

## 0. 設計原則

- **薄殼**：UI 不重寫邏輯，全走 `builder`。
- **源頭驅動**：焊口從「打數字」改回「從這張圖號的現有焊口挑」。
- **緊湊可收**：小螢幕能用——主工作區 + 可收合側欄 + 分頁 + 捲動。
- 全中文、套既有 `theme`、每步有導引。

## 0.5 設計定案（2026-06-24 已對齊）— 取代下方 §1/§4 的 Qt 假設

**技術**：丟 PyQt6 → **HTML/CSS/JS 跑在 pywebview 原生視窗**（非瀏覽器）。橋 `control/co_bridge.py` 已建已驗、引擎不動。

**版面 ＝ 品牌 app-shell**（不是 QTabWidget 平鋪）：
- 左側**品牌藍導航列**：logo ＋「智源工程・修改單系統」＋ 四步驟（① 基本 ② 焊口 ③ 照片圖面 ④ 材料）＋ 底部圖號/連線狀態。
- 右側**一次只顯示一段**內容（順手解掉「太長/小螢幕」）。
- 底部固定**狀態列 ＋ 存草稿/正式建立**。

**視覺語言**：
- 品牌藍 `#1860ab`（取自 logo）＝側欄底色 ＋ 主要動作 ＋ 重點數字（焊口碼用 mono 放大）。
- **淺色企業風**（白底、品牌藍骨架），配藍底白字 logo；深色變體備而不用。
- **用留白與層次分區，不要把每塊都框起來**——少框、多呼吸 ＝ 現代感來源（避免 CTk 那種破敗感）。
- 選中態淡藍底；狀態用語意色（待補=amber、完整=green）。
- **logo**：實際圖檔放在 `ref/`，header 直接用它（沒有才退回佔位圓徽）。
- **公司領域**：智源工程做 **管線 / 鋼構 / 桶槽**——設計可帶這三種工程意象（線稿、克制、品牌藍），但工具本身聚焦管線焊口。
- **生成意象（選用、克制）**：Codex 可用生圖工具產**少量、扁平線稿、品牌藍**的圖，用在空狀態（如「尚未加入焊口」放一張管線焊口線稿）、步驟小圖示等；**不要**滿版背景 / 漸層 / 花俏裝飾——乾淨 app-shell 優先，意象只是點綴。生成圖放 `control/co_wizard_web/img/`。

**仍適用**：§2–§3（焊口源頭挑/照片/材料）、§5（完整度）、§6（對照）、§7（優先序）的**內容**不變，只是把「Qt 怎麼排」換成「HTML app-shell 怎麼排」。§1 的 ASCII 版面、§4 的小螢幕策略以本節為準。

## 2. 主工作區（分頁）

### ① 基本資料
流水號、日期、原因。流水號一填 → 側欄「歷史」與 ② 的焊口清單**立即載入**。
導引：「填流水號（＝圖號），下面會自動帶出這張圖的焊口與歷史」。

### ② 焊口（源頭驅動 — 這是重點修正）
- **上半：「這張圖號的現有焊口」清單**（從管制表 `lookup` 載入：焊口號 / 尺寸 / 材質 / 厚度 / 型式），可搜尋過濾。
- **既有重焊**：在清單**選一筆** → 選操作（裁 / 加 / 縮）→「加入修改」→ 自動算 `2a / 2b…`、**規格自動帶入**。**不再叫你憑空打 base。**
- **新焊口**：旁邊獨立區，選操作 + 填規格 →「加入」→ 自動 `1000+`。
- **下半：「本單焊口」表**（算好的 code / 規格 / 來源）+ 移除。
- 導引：「既有的從上面挑、原圖沒有的用『新焊口』；編號系統自動算」。

### ③ 照片與圖面
加 before / after（可多張）、選圖面 PDF。選好的東西即時進側欄「已選預覽」。
導引：「修改前拍問題、修改後拍完成，附對應圖號 PDF」。

### ④ 材料
結構化材料表單 + 清單（沿用既有欄位）。

## 3. 側欄（可收，補回舊精靈的好東西）

- **歷史**：流水號一填 → 列出這張圖號**過去的修改單**（日期 / 焊口 / 縮圖 / 開資料夾）。＝源頭驅動的另一半（看這張圖做過什麼）。
- **已選預覽 + 標註**：選的照片 / PDF 縮圖；**在圖上圈 原因 / 改什麼 / 材料（重接既有 `gui_annotator`）**——你說的「核心工作所在地」。
- **staging**：staging 資料夾快速匯入照片。

---

## 4. 小螢幕策略（你特別在意的）

- 側欄可**整條收**起 → 只剩主區。
- 主區**分頁**，每頁短；內容多的頁用 `QScrollArea`。
- 預設尺寸不固定超高（約 860×600，可自由縮）。
- 底部 bar 固定，重要動作永遠看得到。

---

## 5. 完整度 / 出單

- 狀態列即時：**完整**（before + after + 圖面PDF 齊）/ **待補**（中文列出還缺什麼）。
- 「正式建立」非完整時擋；「存草稿」一律可。
- 出單走既有 `export_change_order`（建資料夾 + 複製檔 + 寫 record）。
