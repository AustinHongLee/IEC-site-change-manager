## 內測電腦硬條件

- 必須安裝 Microsoft Excel。
- Excel 必須能正常啟動。
- 沒有 Excel 或 Excel COM 不可用的電腦，不允許使用此軟體。
- 不要直接在壓縮檔內執行 exe；一定要先解壓。

## 人工測試步驟

1. 在內測電腦建立一個空白資料夾，例如桌面上的 `IEC_RC0_Test`。
2. 將 `IEC-site-change-manager_0.1.0-alpha_win64_onedir.zip` 解壓到該資料夾。
3. 進入解壓出的 `IEC-site-change-manager` 資料夾。
4. 執行 `IEC-site-change-manager.exe`。
5. 第一次開啟後，確認資料夾內建立：
   `attachments/ records/ output/ pdf/ staging/ logs/ settings.json`
6. 將 repo 內測試附件複製到內測資料夾：
   - `attachments/20260112/0547_AG`
   - `attachments/20250820/55_2a2`
7. 以 CLI 測試 27w 案例：

```powershell
.\IEC-site-change-manager.exe --cli --date 20260112 --force --no-pdf
```

8. 確認產出：
   - `output/20260112/管線修改單_0547_01.xlsm`
   - `records/records.json` 有 1 筆主紀錄、7 筆明細、3 筆材料。
9. 以 CLI 測試 6-slot 案例：

```powershell
.\IEC-site-change-manager.exe --cli --date 20250820 --force --no-pdf
```

10. 確認產出：
    - `output/20250820/管線修改單_0055_01.xlsm`
    - `records/records.json` 新增 1 筆主紀錄與 1 筆明細。
11. 開啟 XLSM，確認照片有貼入、基本欄位可讀。
12. 執行診斷包：

```powershell
.\IEC-site-change-manager.exe --diagnostics
```

13. 確認 `logs/` 或診斷輸出位置可找到 `support_bundle_*.zip`。

## 失敗時回報內容

請回收以下資料：

- 錯誤畫面截圖。
- `logs/app.log`
- `support_bundle_*.zip`
- 測試資料夾內的 `_ERROR.txt`，若有。
- `records/records.json`
- 失敗時使用的附件資料夾。
