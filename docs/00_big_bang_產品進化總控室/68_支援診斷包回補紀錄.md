# 支援診斷包回補紀錄

日期：2026-06-22

## 背景

公司級 EXE 交付後，使用者回報問題不能只靠截圖或口述。需要一個只讀、可重複、低敏感度的診斷包，讓工程端能快速看到版本、啟動守門、資料一致性與輸出能力摘要。

## 本次新增

- `control/diagnostics.py`
  - 產生 `support_bundle_YYYYMMDD_HHMMSS.zip`
  - 內含：
    - `diagnostics.json`
    - `health_check.txt`
  - 不打包 attachments、records 原始資料。
- `tools/collect_support_bundle.py`
  - 開發環境可直接用 CLI 產生診斷包。
- `control/main.py`
  - 新增 `--diagnostics`
  - 新增 `--diagnostics-output`
- `tests/test_diagnostics.py`

## 使用方式

```powershell
python control\main.py --diagnostics
```

指定輸出：

```powershell
python control\main.py --diagnostics --diagnostics-output staging\support_bundle
```

工具版：

```powershell
python tools\collect_support_bundle.py --project-root . --output staging\support_bundle
```
