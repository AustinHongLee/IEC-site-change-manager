# 76 P0-3 打包後真輸出冒煙閉環

日期：2026-06-22

## 結論

Checkpoint G 的 P0-3 已補上可執行 gate：現在 release package 可以在不污染 `dist/IEC-site-change-manager/` 的前提下，複製一份 onedir package 到暫存區，初始化專案資料夾，塞入指定 `attachments/<date>/<folder>`，再用打包後的 `IEC-site-change-manager.exe --cli` 真的產出一張修改單。

這次真 smoke 也抓到一個原本 health-check / diagnostics 都抓不到的問題：Excel VBA 巨集呼叫字串不應帶模組前綴。`'template_file.xlsm'!PDF_匯出.InsertAndFitPicture_ByAddr` 會失敗；改成 `'template_file.xlsm'!InsertAndFitPicture_ByAddr` 後，6-slot 與 27w package smoke 都通過。

## 已落地

- 新增 `tools/run_packaged_cli_smoke.py`
  - 複製 package 到暫存工作區。
  - 先執行 `--repair-project --health-check` 讓 exe 自己建立 `attachments/ records/ output/ pdf/ staging/ logs/ settings.json`。
  - 複製測試附件案例。
  - 執行 `--cli --date <date> --force --no-pdf`。
  - 驗證 `output/<date>/*.xlsm`、`records/records.json` 主表與明細。
- `tools/check_release_package.py`
  - 新增 `--run-cli-smoke`。
  - JSON 結果新增 `cli_smoke` 區塊。
  - CLI smoke 失敗會轉成 package gate error，不再假綠燈。
- `tools/build_release.py`
  - 新增 packaged CLI smoke 串接，可做 build/package/health/resource/true-output 一條龍 gate。
  - 後續因公司版 Excel mandatory，已改為預設執行；快速結構檢查才使用 `--no-cli-smoke`。
- 修正資料路徑
  - `record_manager.py` 的 records/billing/dwg map 改走 `resources.project_path()`，避免打包後寫到 `_internal/records`。
  - `log_config.py` 的 logs 改走 `resources.project_path()`。
  - `weld_control.py` 的 records 查詢改走 `resources.project_path()`。
- 修正 CLI `--no-pdf`
  - 不要求不存在的 PDF 通過完整性檢查，只驗 XLSM。
- 修正 Excel 巨集呼叫
  - 改成呼叫 workbook scope 的 `InsertAndFitPicture_ByAddr`。
