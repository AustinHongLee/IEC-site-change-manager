# 輸出中心命名收斂 P0 小步

日期：2026-06-22

## 背景

Opus 在 53 計畫書指出，GUI 中仍殘留 `btn_photo_grid_pdf` 與 `_export_real_attachments_showcase` 等舊命名，容易讓後續 AI 以為正式輸出中心仍是照片 PDF 或 showcase/demo 流程。

本次只做低風險命名收斂，不重排 UI、不改輸出流程、不移除 demo CLI。

## 本次調整

- `RecordManagerPanel._build_ui()`
  - local 變數 `btn_photo_grid_pdf` 改為 `btn_output_center`。
  - 按鈕顯示文字仍維持「輸出中心」。
- `RecordManagerPanel._choose_output_center_scope()`
  - dialog 標題由「照片 PDF 輸出」改為「輸出中心」。
- 移除未被程式使用的舊 alias：
  - `_export_real_attachments_showcase = _export_site_output_center`

## 刻意保留

以下相容 alias 暫時保留：

- `_showcase_report_keys`
- `_showcase_scope_options`
- `_format_showcase_content_label`
- `_normalize_showcase_output_dir`
- `_showcase_output_items`
- `_format_showcase_export_confirmation`
- `_format_showcase_export_message`

原因：這些 helper 仍有舊測試覆蓋，且不影響正式 UI 文案。等下一輪把測試呼叫點改到 `_output_center_*` 後，再逐步拔除。

## 驗收

確認：

- 面板上仍存在「輸出中心」入口。
- 面板不再暴露 `_export_real_attachments_showcase` 舊入口。
