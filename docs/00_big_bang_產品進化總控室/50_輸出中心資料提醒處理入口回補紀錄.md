# 輸出中心資料提醒處理入口回補紀錄

日期：2026-06-18

## 目的

上一階段資料提醒已可定位修改單，也能在定位失敗時自動解除篩選。

本階段讓資料提醒依 `code` 提供更精準的處理入口，讓使用者不只知道哪筆有問題，也知道下一步該做什麼。

## 調整內容

### `control/gui_panels.py`

輸出中心結果 dialog 新增：

- `處理提醒` 按鈕

資料提醒列新增 `issue_action`：

| issue code | 動作 |
|---|---|
| `note` | 開啟附件資料夾補 `note.txt` |
| `before_photo` | 開啟附件資料夾補 before 照片 |
| `after_photo` | 開啟附件資料夾補 after 照片 |
| `parse_error` | 開啟附件資料夾檢查 `GroupWeld.txt` / `materials.txt` / 資料夾命名 |
| `weld_or_material` | 定位修改單檢查焊口或用料資料 |
| 其他 | 定位修改單 |

處理流程：

1. 優先用資料提醒的 `record_ref` 找目前紀錄列
2. 找不到時沿用上一階段自動解除篩選並重試
3. 若動作是開資料夾，會優先用 record 的 `folder_path`
4. 若 record 不存在但 `date/folder` 存在，直接開 `attachments/date/folder`
5. 若仍找不到，顯示資料夾不存在提示
