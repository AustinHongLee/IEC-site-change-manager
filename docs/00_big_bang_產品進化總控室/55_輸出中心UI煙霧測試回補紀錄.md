# 輸出中心 UI 煙霧測試回補紀錄

日期：2026-06-22

## 背景

Opus 指出「輸出中心」目前還是 `RecordManagerPanel` 裡的一顆按鈕與結果彈窗，但產品方向已經把它視為現場交付閉環核心。若後續要升級成正式分頁或工作台，必須先有最小 UI safety net，避免改 UI 巨石時看不到入口是否被誤傷。

## 本次新增

新增 `tests/test_output_center_ui_smoke.py`：

1. 建立 `RecordManagerPanel`，確認面板上仍存在「輸出中心」入口按鈕。
2. 建立輸出中心結果 dialog 的 smoke test，確認：
   - dialog 標題為「輸出中心結果」。
   - 結果樹 `QTreeWidget` 可建立。
   - 「開啟選取檔案」「定位修改單」「處理提醒」「開啟輸出資料夾」等關鍵操作按鈕存在。

此測試只檢查 UI 骨架，不觸發真實輸出、不開啟檔案、不寫 records。

## 驗收

已執行：

```powershell
python -m pytest -s -q tests/test_output_center_ui_smoke.py
```

結果：

```text
2 passed
```
