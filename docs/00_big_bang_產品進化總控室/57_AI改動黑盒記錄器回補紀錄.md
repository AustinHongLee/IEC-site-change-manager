# AI 改動黑盒記錄器回補紀錄

日期：2026-06-22

## 背景

Opus 在 `53_下一步計畫書_輸出中心閉環與EXE產品化.md` 建議加入「AI 改動黑盒記錄器」：每次 Codex 或其他 AI 跑完，應留下 `git diff --stat`、測試摘要與 health-check 摘要，方便事後追蹤哪一次改動造成問題。

## 本次新增

- `tools/record_ai_session.py`
  - 將 AI 工程工作摘要寫入 `docs/00_big_bang_產品進化總控室/ai_sessions/`。
  - 寫入前抓取 git snapshot：
    - `git status --short --branch`
    - `git rev-parse --short HEAD`
    - `git log --oneline -5`
    - `git diff --stat`
  - 支援 `--summary`、`--validation`、`--risk`、`--next` 多筆 bullet。
  - 支援 `--dry-run` 預覽。
  - 檔名撞名時自動加 `-2`、`-3`。
- `tests/test_record_ai_session_tool.py`
  - 驗證 topic slug、日期注入、檔名防撞、markdown 結構。

## 使用方式

範例：

```powershell
python tools/record_ai_session.py `
  --topic "P0 guardrails" `
  --goal "建立 repo 護欄與 UI smoke safety net" `
  --summary "新增 repo_guard 與 pre-commit hook" `
  --validation "python -m pytest -s .\tests：345 passed" `
  --validation "python .\control\main.py --health-check：healthy" `
  --risk "health audit 仍有 attachments-only 測試資料 warning" `
  --next "輸出中心從彈窗升級成正式分頁前，先補更多 UI smoke"
```

## 驗收

已執行：

```powershell
python -m pytest -s -q tests/test_record_ai_session_tool.py
```

預期結果：

```text
4 passed
```

## 注意

此工具只產生工程紀錄，不取代 git commit、測試或人工 review。若 session note 本身也要進 repo，仍需正常走 `repo_guard`、commit、push。
